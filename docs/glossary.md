# Glossary

This glossary provides beginner-friendly definitions of key terms used in the project.

## Agent
An **Agent** is a specialized, autonomous component designed to perform a specific task. Agents can use tools, analyze data, and report back results.

## Code Gen Module
A **Code Gen Module** is the pluggable unit that handles LLM-powered code generation.

## Context file
A **Context file** is a structured input containing a file path, content, and source label (e.g., workspace, agent-memory) that provides relevant code and historical data to help agents make informed decisions.

## Domain
A **Domain** represents a specific CHT (Community Health Toolkit) functional area. Examples of domains include contacts, forms-and-reports, tasks-and-targets, messaging, and more.

## Human validation checkpoint
A **Human validation checkpoint** is a point in an automated process where the workflow pauses to allow a human user to review, verify, and approve the generated work before continuing.

## Knowledge base
A **Knowledge base** is a collection of context files that act as an agent-memory system. It serves as an internal library of resources that agents can query to understand previous issues.

## LangGraph
**LangGraph** is the framework used in this project for building complex, stateful agent workflows.

## Layer
A **Layer** is a modular boundary in the architecture that encapsulates a specific capability (e.g., code generation) behind a standard interface, allowing different implementations to be swapped via a registry.

## LLM Provider
An **LLM Provider** (Large Language Model Provider) is an external service providing access to language models, which agents use to understand and generate text.

## MCP
**MCP** stands for **Model Context Protocol**. It is a standard for AI tool integration, ensuring smooth communication between models, tools, and data sources.

## Orchestration plan
An **Orchestration plan** is a step-by-step roadmap generated by an agent detailing exactly how a specific development ticket or request should be implemented.

## Registry
A **Registry** is a pattern used to select module implementations at runtime (e.g.  CodeGenModuleRegistry).

## Supervisor
A **Supervisor** is a higher-level agent responsible for orchestrating, delegating tasks to, and managing multiple specialized agents in a workflow.

## Staging
**Staging** is the preview mode where generated files are written to a temp directory before applying to cht-core.

## Token
A **Token** is a basic unit of text used by Language Models to process language. LLM APIs often bill based on token usage, and models have limits on token processing.

## Workflow
A **Workflow** is the complete sequence of operations, from an initial input to the final output, often involving multiple agents and tools.
