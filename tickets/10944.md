# Description

`api/src/services/messaging.js` assumes every row returned from `db.medic.allDocs({ include_docs: true })` contains a `doc`. That assumption is not always true. CouchDB/PouchDB can omit `doc` for `not_found`, deleted, or race-condition rows. When this happens, `updateMessageTaskStates` passes `undefined` entries into task lookup logic and throws a runtime exception:

```
TypeError: Cannot read properties of undefined (reading 'tasks')
```

## Affected Code

- `api/src/services/messaging.js`
  - `updateMessageTaskStates` (around line 281+)
  - `getTaskForMessage` (around line 29)
  - `getTaskAndDocForMessage` (around line 34)

Inside `updateMessageTaskStates`, rows are mapped without filtering missing docs:

```js
const docs = results.rows.map(r => r.doc);
const stateChangesByDocId = applyTaskStateChangesToDocs(taskStateChanges, docs);
```

If any row has no `doc`, the call chain below crashes because `doc` is `undefined`.

## Requirements

- `updateMessageTaskStates` must not crash when `allDocs` returns rows missing `doc`.
- Rows without a `doc` are skipped; processing continues for the rows that DO have a doc.
- A warning log entry is emitted noting how many rows were skipped and the affected ids (truncated if very many).
- The helper functions `getTaskForMessage` and `getTaskAndDocForMessage` guard against undefined inputs to prevent secondary crashes in the same code path.

## Acceptance Criteria

- Calling `updateMessageTaskStates` with an `allDocs` result containing one or more rows whose `doc` is `undefined` does NOT throw.
- The function returns successfully and the state changes for the rows that had docs are applied as before.
- A new unit test in the messaging service spec covers the "rows without doc" scenario and asserts no throw + correct processing of the rows that did have docs.
- Existing tests in the messaging service spec continue to pass.

## Constraints

- Behavior for the all-rows-have-doc case must be unchanged.
- No new dependencies.
- Fix is in `api/src/services/messaging.js` and its tests.

## Technical Context

**Components:**
- `api/src/services/messaging.js`
- `api/tests/mocha/services/messaging.spec.js`

## Suggested Fix

Filter out rows without a `doc` before mapping, and add a warning log when any are skipped:

```js
const skipped = results.rows.filter(r => !r.doc);
if (skipped.length) {
  logger.warn(`updateMessageTaskStates: ${skipped.length} row(s) missing doc; skipping`);
}
const docs = results.rows.filter(r => r.doc).map(r => r.doc);
const stateChangesByDocId = applyTaskStateChangesToDocs(taskStateChanges, docs);
```

Apply analogous undefined-guards to `getTaskForMessage` and `getTaskAndDocForMessage`.
