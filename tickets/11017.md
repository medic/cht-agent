# Description

When the user has the `can_view_old_navigation` permission they can see the old-style header tabs. There are currently a couple of style issues with how the new UI Extensions are rendered for the legacy UI.

## 1. Add `accent_color` to the "tab" part of the header for `app_main_tab` extensions

When a main header tab is selected, the "tab" part of the header should be highlighted with the proper `accent_color`. For UI Extensions, this background-color is not being set. With `"accent_color": "red"` configured for the extension, the red color should also be applied as the background around the extension's tab name (e.g. "Hello Extension"), matching the behavior of built-in tabs.

## 2. Do not render the extra header bar for `app_drawer_tab` extensions

The existing pages reachable from the legacy hamburger menu (About, Training Materials, etc.) do not render the extra colored tool-bar space at the top of the page that you get with the main header tabs. When opening an `app_drawer_tab` extension from the legacy hamburger menu, the extra header bar is still rendered. It should not be rendered for `app_drawer_tab` extensions.

## Requirements

- `app_main_tab` UI Extensions render with the configured `accent_color` as the background of their selected tab in the legacy header, identical to built-in main tabs.
- `app_drawer_tab` UI Extensions opened from the legacy hamburger menu do not render the extra colored header tool-bar above the page content.
- No change to the rendering of built-in legacy tabs or to the new (non-legacy) UI extension surfaces.

## Acceptance Criteria

- With `can_view_old_navigation` permission granted and a UI Extension configured with `"slot": "app_main_tab"` and `"accent_color": "<color>"`, the selected tab in the legacy header shows the configured accent color as its background.
- With the same permission, opening an `app_drawer_tab` extension from the legacy hamburger menu displays the page directly under the existing hamburger header — no extra colored tool-bar inserted.
- Existing About and Training Materials pages render unchanged.
- No regression for `can_view_old_navigation`-off users.

## Constraints

- Only affects the legacy UI styling path. The non-legacy UI rendering must be untouched.
- Use the existing accent-color application path for built-in tabs; do not duplicate styling logic.
- Compatible with the current UI Extensions configuration schema.
